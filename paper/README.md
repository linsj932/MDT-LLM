## Writing workflow

1. **While experiments run:** draft all sections except Results (done above)
2. **When judge scoring completes:** run `analyze_results.py` to produce figures + tables
3. **Fill in `08_results.md`** by reading the numerical outputs of the analysis script
4. **Fill in `[TBD]`** placeholders in Discussion with specific findings from Results
5. **Consolidate:** paste all `.md` files in order into an Overleaf LaTeX project (elsarticle template)

## Style notes

- **Tense:** Past tense for experiments performed; present tense for persistent facts.
- **Citations:** numerical `[1]` style per Elsevier `elsarticle-num`.
- **Equations:** inline with `$...$`, display with `$$...$$`; converted to LaTeX on Overleaf.
- **Figures:** Figure 1 is architecture (Mermaid → export SVG); Figures 2–7 are generated by `analyze_results.py` as PNG at 300 dpi.
- **Tone:** technical/methodological, not clinical/humanistic. JKSUCIS reviewers are CS-background.
